
# **Práctica de Debugging en Java**

Este repositorio contiene ejemplos diseñados para practicar el uso del depurador en programación Java. A través de estos ejercicios, identificarás y resolverás errores lógicos en el código, comprenderás el flujo de ejecución y mejorarás tus habilidades de análisis.

---

## **Por qué es importante el Debugging**
El debugging es una habilidad esencial en programación porque permite:
- **Detectar errores:** Identificar problemas en el código, como errores lógicos o cálculos incorrectos.
- **Entender el flujo de ejecución:** Ver cómo se ejecuta el programa paso a paso.
- **Ahorrar tiempo:** Resolver problemas de manera eficiente y evitar errores repetitivos.
- **Aprender del proceso:** Conocer mejor el comportamiento del programa y mejorar tus habilidades de programación.

---

## **Principales características del Depurador**
Los entornos de desarrollo integrados (IDEs) como **Eclipse** y **Visual Studio Code** ofrecen herramientas de depuración muy potentes, entre ellas:
- **Breakpoints:** Pausar la ejecución en líneas específicas para analizar el estado del programa en ese momento.
- **Ejecución paso a paso:** Avanzar línea por línea para seguir el flujo del programa.
- **Inspección de variables:** Ver los valores actuales de las variables en tiempo real.
- **Pila de llamadas:** Revisar qué funciones se están ejecutando y en qué orden.
- **Condiciones en breakpoints:** Establecer criterios específicos para pausar la ejecución solo bajo ciertas condiciones.

## **Archivos del Repositorio**
Este repositorio incluye los siguientes archivos Java con errores intencionales que te ayudarán a practicar debugging:



1. **`DebugExample1.java`**
   - Un bucle que lanza excepciones debido a un error en la condición de iteración.
   - Perfecto para analizar y corregir errores en límites de arrays.

2. **`DebugExample2.java`**
   - Un programa con una condición lógica incorrecta que genera resultados inesperados.
   - Ideal para trabajar con estructuras de control y depuración paso a paso.

3. **`ArraySum.java`**
   - Contiene errores relacionados con el manejo de índices en cálculos con arrays.
   - Te ayudará a practicar la lógica de los bucles y el acceso a arrays.

4. **`Factorial.java`**
   - Implementa el cálculo recursivo de factoriales con un error lógico en los casos base.
   - Diseñado para que analices cómo funcionan las llamadas recursivas y resuelvas problemas como el `StackOverflowError`.

## **Cómo empezar**
1. **Clonar el Repositorio:**
   ```bash
   git clone <URL_DEL_REPOSITORIO>
   cd <NOMBRE_DEL_REPOSITORIO>
   ```
2. **Importar los Archivos:**
   - Abre los archivos en tu IDE favorito, como Eclipse o Visual Studio Code.

3. **Practicar Debugging:**
   - Configura breakpoints y utiliza las herramientas de depuración para analizar cada programa.
   - Identifica los errores, corrige el código, y verifica que los programas funcionen correctamente.
